# Contributing Guide

Thank you for considering contributing!

## How to Contribute
- Fork the repository and create a feature branch
- Make your changes with clear, well-commented code
- Add or update tests as needed
- Update documentation if your change affects usage or APIs
- Open a pull request with a clear description and screenshots for UI changes

## Code Style
- Use TypeScript and Svelte best practices
- Prefer small, focused commits
- Keep functions and components single-purpose
- Run `npm run check` and `npm run test` before submitting

## Reporting Issues
- Use GitHub Issues for bugs, feature requests, or questions
- Include steps to reproduce and expected vs actual behavior

## Code of Conduct
- Be respectful and constructive
- See CODE_OF_CONDUCT.md for details (if present)
